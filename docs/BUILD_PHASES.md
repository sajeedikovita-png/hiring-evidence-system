# Build Phases

## Phase 1: Planning Documents

Create all Markdown planning documents.

No app coding yet.

## Phase 2: Design System

Build reusable UI components:

- Page layout
- Sidebar
- Header
- Cards
- Tables
- Buttons
- Badges
- Forms
- Warning cards
- Notes box

## Phase 3: Recruiter Job Flow

Build:

- Company dashboard
- Create job page
- Job criteria review page
- Publish application link

## Phase 4: Candidate Application Flow

Build:

- Public application landing page
- Resume upload page
- Profile links page
- Questionnaire page
- Consent page
- Submit confirmation

## Phase 5: AI Processing

Build:

- Job analyzer
- Question generator
- Resume parser
- Evidence matcher
- Fairness checker
- Report generator

Use mock AI first if needed.

Then connect real AI API.

## Phase 6: Candidate Report

Build:

- Candidate overview
- Evidence summary
- Evidence matrix
- Missing evidence
- Questionnaire review
- Suggested interview questions
- Fairness check
- Recruiter notes
- Human decision section

## Phase 7: Audit and PDF

Build:

- Decision log
- Audit log
- PDF export

## Phase 8: Pilot

Test with:

- 3 sample jobs
- 10 sample candidates
- 3 recruiter users

Goal:

Recruiter must understand the report within 2 minutes.
Recruiter must feel the report is useful enough to use in real hiring.
