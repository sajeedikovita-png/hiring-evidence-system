# Design System

## Design Direction

The design must feel:

- Minimal
- Professional
- Authentic
- Calm
- Trustworthy
- Human-centered
- Evidence-focused

The product must not feel like a flashy AI tool.

## Visual Style

Use:

- Warm off-white or neutral background
- Deep charcoal text
- Muted slate, soft blue, or soft green accents
- Clean cards
- Subtle borders
- Soft shadows only when needed
- Generous spacing
- Clear tables
- Professional typography

Avoid:

- Robot icons
- Sparkles
- Neon gradients
- Cyber AI style
- Heavy glassmorphism
- Overly playful colors
- Overdesigned animations

Recommended:

- Inter
- Geist
- Manrope

## Component Rules

### Buttons

Primary buttons should be clear but not flashy.

Use labels like:

- Create job
- Analyze requirements
- Review report
- Save decision
- Export PDF

Avoid labels like:

- Let AI decide
- Find best candidate
- Auto reject
- Magic match

### Cards

Use cards for:

- Dashboard metrics
- Candidate summaries
- Evidence sections
- Fairness warnings
- Recruiter notes

### Tables

Use tables for:

- Job criteria
- Candidate lists
- Evidence matrix
- Audit logs

Tables must be readable and not crowded.

### Status Badges

Use badges such as:

- Strong evidence
- Needs verification
- Missing evidence
- Human review required
- Report ready
- Decision saved

Avoid badges such as:

- Best candidate
- Weak candidate
- AI rejected
- Perfect match
